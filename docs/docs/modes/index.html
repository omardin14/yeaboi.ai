yeaboi

Modes

Everything a lead does between planning and delivery — six modes, one command.

Planning

Turn a project description into epics, stories, tasks, and a sprint plan — then push straight to Jira or Azure DevOps. A five-stage pipeline (analyzer → feature generator → story writer → task decomposer → sprint planner) runs with a human-in-the-loop accept/edit/reject checkpoint after every stage, so nothing ships without a review.

Full Planning docs →

Daily Standup

Detect what everyone shipped, score sprint progress, and deliver the summary — on a schedule, even when yeaboi is closed. Activity is pulled from Jira, Azure DevOps, GitHub, Confluence, and local git; an OS-native scheduler (launchd/crontab) fires the run ahead of your actual standup time.

Full Daily Standup docs →

Retro

Open a board your whole team joins from any browser on the network. Cards, reactions, a shared timer — then AI action items. A small local HTTP server hosts the board with a share code and QR join, no install required for teammates.

Full Retro docs →

Performance

Prep and write up 1:1s, and synthesize six-month reviews from each engineer's real delivery history. The roster is derived from who actually did the work in Jira/Azure DevOps, and open action items from one 1:1 carry forward automatically into the prep for the next.

Full Performance docs →

Reporting

Turn a sprint or a whole quarter of delivered work into a business-ready summary and a slide deck for stakeholders. Choose last sprint, last month, or a whole calendar quarter with a sprint multi-select, and export as Markdown, HTML, or a keyboard-navigable slide deck.

Full Reporting docs →

Team Analysis

Read your team's real velocity and history first, so estimates and capacity are calibrated to how you actually deliver. An analysis profile feeds team size, sprint length, velocity, tech stack, and integrations straight into Planning's Smart intake, cutting the questions you're asked down to the essentials.
